Retour sur le ParisJS Meetup #7 et Ryan Dahl, le créateur de Node.js, sur Paris

Catégorie : Conférence | 26 mai 2011 | Tags :  Fonctionnelle, HTML5, Javascript, NoSQL, Node.js, ParisJS

Cela faisait quelques mois que je n’avais pas pu me rendre à un évènement du Paris JS, une communauté Javascript sur Paris. Et bien j’ai rattrapé ce retard en me rendant hier soir à la 7ème (déjà…) soirée du Paris JS dans les locaux de Vidal (souvenirs…).

Au menu, 5 talks :

  • développement web pour TV connectés et spécifiquement Samsung
  • retour d’expérience intéressant sur CouchDB avec des exemples de MapReduce, les fonctionnalités intéressantes et celles à éviter…
  • HTML5 et vidéo avec un état des specs et quelques implémentations (et ce n’est malheureusement pas pour demain )
  • expressivité en Javascript (non, ce n’est pas CoffeeScript !!!) avec une présentation de la librairie underscore.js, je l’utilise tous les jours donc je ne peux que la recommander car elle permet vraiment d’améliorer la lisibilité du code
  • et enfin un petit point d’avancement du jeu PacMaze

Pour un compte-rendu plus exhaustif, je vous renvoie sur le blog de William et plus particulièrement sur cet article.

Enfin, une belle annonce a été faite avec la venue sur Paris du créateur de Node.JS à savoir Ryan Dahl !

Cela se passera le 8 Juin dans les locaux de Jolicloud. Une seule chose à faire : inscrivez-vous vite !

Play! Framework : nouvelle librairie play.libs.F

Catégorie : Framework | 03 mai 2011 | Tags :  Fonctionnelle, Java, Play

Play! Framework, que je vous avais présenté lors de la sortie de la version 1.1, continue sa route avec une nouvelle version 1.2 (et même 1.2.1 depuis hier) qui apporte moult nouveautés.

Au menu : un système de gestion de dépendances, la gestion des Web Sockets et du Long Polling, un système de gestion d’évolution de la base de données (un must que j’apprécie déjà !), H2 qui remplace Hsqldb ou bien encore une meilleure intégration du langage Scala (qui fera l’objet d’un prochain article).

Mais une autre nouveauté, peu mise en avant, a fait son apparition par une librairie. Ainsi, aux côtés des très connues play.libs Files, Mail ou WS se trouve play.libs.F ou la librairie fonctionnelle de Play. Regardons cela de plus près…

Au menu :

  • Option<T>
  • Either<A, B>
  • Tuple<A, B>
  • Promise<V>

Java + Programmation fonctionnelle + Funky = FunkyJFunctional !

Catégorie : Framework | 05 avril 2011 | Tags :  Fonctionnelle, Guava, Java

Courant de semaine dernière, alors que je faisais un important refactoring pour améliorer la lisibilité du code, je suis tombé sur une librairie au nom assez rigolo : FunkyJFunctional. Comme je suis un Funky développeur  j’ai regardé de plus près…

Comme vous le savez peut-être déjà, j’aime beaucoup la programmation fonctionnelle et le langage Scala. Et, sur ce refactoring qui concernait à la fois du code métier mais aussi des tests (tout ça en Java), j’ai pu mettre en oeuvre et utiliser de manière poussée des librairies comme Google Guava, Fest-Assert ou bien encore Mockito et plus particulièrement son package BDDMockito (pour faire du given-when-then). Cela pourra d’ailleurs faire l’objet d’un prochain article avec un exemple de code avant-après, le résultat est assez bluffant.

Bien sûr, ce n’est pas encore du DSL comme on peut en voir en Scala mais à l’aide de quelques astuces syntaxiques, ces librairies nous offrent une API fluent qui permet réellement de lire du code comme si nous lisions une phrase. Ceci est un aspect très important surtout pour les tests qui pour moi reste la documentation principale du code d’une application.

Et donc, pour pousser encore plus loin ce concept de lisibilité, voici une petite librairie qui simplifie encore plus (oui oui) certaines API Java (Runnable, Comparator), Google Guava, Swing et même Wicket ! Alors, le Funky qui est en vous est intéressé ?
